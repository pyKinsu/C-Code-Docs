# Contributing

Thank you for your interest in contributing! 🎉  
This project aims to provide a **beginner-friendly guide to C programming**, and your contributions help make it better for everyone.

---

## How You Can Contribute

You can contribute in many ways:

- Fix grammar or spelling mistakes  
- Improve explanations for clarity  
- Add better or simpler examples  
- Report bugs or incorrect information  
- Suggest new topics or chapters  

> [!TIP]
> Even small improvements (like fixing a typo) are valuable.

---

## Getting Started

1. Fork the repository  
2. Clone your fork  
3. Create a new branch  

```bash
git checkout -b feature/your-change-name
````

4. Make your changes
5. Commit your changes

```bash
git commit -m "Add: improved explanation for variables"
```

6. Push to your fork

```bash
git push origin feature/your-change-name
```

7. Open a Pull Request

> [!NOTE]
> Use clear and descriptive commit messages so reviewers understand your changes easily.

---

## Contribution Guidelines

* Keep content **simple and beginner-friendly**
* Follow the existing **writing style and structure**
* Use proper formatting (headings, lists, code blocks)
* Ensure code examples are correct and runnable
* Check that all links work properly

> [!TIP]
> Before submitting, preview your changes on GitHub to ensure formatting looks correct.

---

## Reporting Issues

If you find a problem, open an issue and include:

* What is wrong
* Where it is located (file/section)
* Suggested fix (if possible)

> [!NOTE]
> Clear issue descriptions help resolve problems faster.

---

## Code of Conduct

* Be respectful and constructive
* Avoid spam or irrelevant changes
* Help others learn and grow

---

## Thank You ❤️

Your contribution helps make this project a better learning resource for everyone.
